# Contributing to Argument Architect

Thank you for your interest in improving Argument Architect. This project is meant to support teachers and students doing the hard work of learning to argue in writing, and contributions that make that easier, clearer, or more accurate are genuinely welcome.

## Ways to contribute

- **Report bugs.** Open an issue and include the browser you were using, what you expected to happen, and what actually happened. A short screen recording is ideal but not required.
- **Suggest improvements.** Rubric alignment, clearer scaffolds, better prompts, and accessibility fixes are all high value.
- **Contribute code.** Fork the repo, create a feature branch, and open a pull request against `main`.
- **Contribute prompts or exercises.** Curriculum-aligned additions are welcome. Please note the target course (AP Language, first-year composition, etc.) and any source attribution.

## Pull request guidelines

1. Fork the repository and create a branch from `main` named after the change (for example, `feature/synthesis-prompts` or `fix/mobile-layout`).
2. Keep changes focused. One PR should address one thing.
3. Test in at least one modern browser (Chrome, Safari, or Firefox) before opening the PR.
4. Update the README if your change affects how users interact with the app.
5. In your PR description, explain what the change does and why it helps learners or educators.

## Style

- Keep the app dependency-free when possible.
- Prefer semantic HTML and accessible patterns (labels on inputs, keyboard-friendly controls, adequate contrast).
- Write JavaScript that a motivated student could read and learn from.

## Code of Conduct

By participating, you agree to the terms in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Questions

If you are unsure whether an idea fits, open an issue to discuss before writing code. Teachers especially: please do not worry about writing "polished" issues. A clear description of what would help your students is plenty.
